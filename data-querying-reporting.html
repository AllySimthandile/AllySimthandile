Data Querying & Reporting Project

In this project, I began with an Excel-based data profiling task using Power Query. The goal was to understand the structure and quality of data to determine if there were potential issues such as fraud, data duplication, or anomalies in transactional data. This initial profiling step helped in making decisions about whether further forensic analysis was required.

After profiling the data, I moved to a more advanced level of analysis using SQL within Microsoft Access. I performed a complete forensic analysis on invoice data. To enhance usability, I created a Switchboard interface that allows users to:

  • Run forensic queries (pre-built SQL queries)
  • View analytical reports
  • Enter new invoice data through forms
This user-friendly Access dashboard makes the system accessible to non-technical users while retaining the depth of forensic insight.

This project demonstrates robust skills in data extraction, transformation, and reporting using a combination of Excel and SQL, with a strong focus on data integrity and user accessibility for business intelligence.

Technologies Used

  • SQL
  • MS Excel (Power Query, PivotTables, VLOOKUP, Macros)
  • MS Access

Key Features

  • Complex data extraction with SQL joins and subqueries
  • Automated Excel reporting templates
  • Data validation and cleansing in Excel
  • Dashboard creation for key metrics in MS Access
